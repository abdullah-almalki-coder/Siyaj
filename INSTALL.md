## خطوات التثبيت

### 1. تحميل الملفات

```bash
# إذا كان المشروع على GitHub
git clone https://github.com/your-username/siyaj.git
cd siyaj

# أو انسخ المجلد يدوياً إلى Linux ثم انتقل إليه
cd /path/to/siyaj
```

### 2. تثبيت المكتبات

```bash
pip3 install -r requirements.txt
```

أو إذا واجهت مشكلة صلاحيات:

```bash
pip3 install --user -r requirements.txt
```

### 3. التحقق من التثبيت

```bash
python3 -c "import colorama; print('colorama OK')"
```

## تشغيل الأداة

> **يجب تشغيل الأداة بـ sudo** — بدونه لا يمكن قراءة `/etc/shadow` و`sudoers`

```bash
# فحص كامل (SSH + Firewall + Users)
sudo python3 main.py --audit all

# فحص SSH فقط
sudo python3 main.py --audit ssh

# فحص الجدار الناري فقط
sudo python3 main.py --audit firewall

# فحص المستخدمين فقط
sudo python3 main.py --audit users

# فحص كامل + تصدير تقرير JSON
sudo python3 main.py --audit all --output json

# فحص كامل + تصدير تقرير TXT
sudo python3 main.py --audit all --output txt
```

## `main.py` — نقطة الدخول

**ماذا يفعل؟**

هو أول ملف يُشغَّل. مهمته:
1. التحقق أن الأداة تعمل بصلاحية root (إذا لا → يخرج فوراً برسالة خطأ)
2. قراءة المعاملات من المستخدم عبر `--audit` و`--output`
3. تحميل قواعد CIS من `cis_rules.json`
4. استدعاء وحدة أو أكثر من وحدات الفحص
5. طلب حساب الدرجات من `scoring.py`
6. عرض النتائج عبر `ui.py`
7. تصدير التقرير إن طُلب

**الدوال الرئيسية:**

| الدالة | الوظيفة |
|--------|---------|
| `load_rules()` | يفتح `cis_rules.json` ويُعيده كـ dict، يخرج بخطأ إذا الملف غير موجود أو JSON تالف |
| `run_audit()` | يُشغّل الفحوصات المطلوبة ويجمع النتائج ثم يُصدّر التقرير |
| `main()` | يتحقق من root، يُهيئ argparse، يُشغّل كل شيء |

## `ssh_audit.py` — فحص SSH

**ماذا يفعل؟**

يقرأ الإعدادات **الفعلية** لـ SSH عبر الأمر `sshd -T` (الذي يُظهر القيم الفعلية بما فيها الافتراضية المدمجة)، ويقارن كل إعداد بالقيمة الآمنة المحددة في `cis_rules.json`. إذا تعذّر `sshd -T`، يرجع لقراءة `/etc/ssh/sshd_config` مباشرة (متتبعاً `Include`)، ويحكم على أي إعداد **غير مكتوب** بقيمته الافتراضية الحقيقية في OpenSSH بدل اعتباره فاشلاً. وإذا لم يوجد ملف SSH أصلاً (SSH غير مثبّت)، يُعطى النطاق `WARN`/«غير منطبق» لا فشلاً.

**الإعدادات التي يفحصها:**

| الإعداد | القيمة الآمنة | الخطر عند المخالفة |
|---------|--------------|-------------------|
| `PermitRootLogin` | no | المهاجم يستطيع تسجيل الدخول مباشرة كـ root |
| `PasswordAuthentication` | no | هجمات brute force على كلمات المرور |
| `Protocol` | 2 (افتراضي في OpenSSH الحديث) | SSH1 به ثغرات خطيرة |
| `X11Forwarding` | no | يسمح بتشغيل تطبيقات رسومية عن بُعد |
| `MaxAuthTries` | ≤ 4 | يحدّ من محاولات تخمين كلمة المرور |
| `IgnoreRhosts` | yes | ملفات `.rhosts` قديمة وغير آمنة |
| `HostbasedAuthentication` | no | تسمح بالدخول بناءً على اسم الجهاز فقط |
| `PermitEmptyPasswords` | no | يمنع الدخول بكلمة مرور فارغة |

**الدوال الرئيسية:**

| الدالة | الوظيفة |
|--------|---------|
| `audit_ssh()` | الدالة الرئيسية، تُعيد قائمة نتائج |
| `_effective_config()` | تقرأ الإعداد الفعلي عبر `sshd -T` (يشمل القيم الافتراضية) |
| `_parse_sshd_config()` | احتياطي: تبدأ قراءة ملف الإعداد عند تعذّر `sshd -T` |
| `_read_config_file()` | تقرأ ملفاً واحداً وتتبع `Include` للملفات الفرعية، تستخدم first-value-wins |
| `SSH_DEFAULTS` | جدول القيم الافتراضية لـ OpenSSH، يُستخدم لتقييم الإعدادات غير المكتوبة |
| `_compare()` | تقارن القيمة الفعلية بالمتوقعة (مساواة أو أصغر/يساوي) |

## `firewall_audit.py` — فحص الجدار الناري

**ماذا يفعل؟**

يتحقق من حالة UFW (جدار الحماية الافتراضي في Ubuntu/Debian) وفحص المنافذ الخطرة المفتوحة.

**الفحوصات الثلاثة:**

| الفحص | الأمر المستخدم | الخطر عند الفشل |
|-------|---------------|----------------|
| UFW نشط؟ | `ufw status` | لا يوجد جدار حماية أصلاً |
| السياسة الافتراضية = deny؟ | `ufw status verbose` | كل الاتصالات الواردة مسموح بها |
| منافذ خطرة مفتوحة؟ | `ss -tlnp` | خدمات قديمة أو خطرة مكشوفة للإنترنت |

**المنافذ الخطرة التي يفحصها:**

| المنفذ | الخدمة | الخطر |
|--------|--------|-------|
| 21 | FTP | نص صريح، بدون تشفير |
| 23 | Telnet | نص صريح، اُستبدل بـ SSH |
| 135, 139, 445 | Windows SMB | ثغرات EternalBlue وWannaCry |
| 512, 513, 514 | rexec/rlogin/rsh | خدمات Unix قديمة وغير آمنة |
| 3389 | RDP (Windows) | هجمات brute force شائعة |
| 5900 | VNC | بدون تشفير افتراضياً |

**الدوال الرئيسية:**

| الدالة | الوظيفة |
|--------|---------|
| `audit_firewall()` | تُشغّل الفحوصات الثلاثة وتُعيد النتائج |
| `_run()` | تُنفّذ أمر shell تحت `LC_ALL=C` (لضمان مخرجات إنجليزية ثابتة) وتُعيد الناتج |
| `_ufw_active()` | تتحقق من `Status: active` في ناتج ufw |
| `_default_deny()` | تتحقق من السياسة الافتراضية |
| `_open_dangerous_ports()` | تستخدم regex للبحث عن المنافذ الخطرة في ناتج `ss` |

## `users_audit.py` — فحص المستخدمين

**ماذا يفعل؟**

يفحص حسابات المستخدمين في النظام للكشف عن إعدادات خطرة.

**الفحوصات الأربعة:**

| الفحص | الملف المُقرأ | الخطر عند الفشل |
|-------|-------------|----------------|
| root مقفل؟ | `/etc/shadow` أو `passwd -S root` | المهاجم يستطيع تسجيل الدخول كـ root محلياً |
| حسابات UID=0 غير root؟ | `/etc/passwd` عبر مكتبة `pwd` | حساب مخفي بصلاحيات كاملة |
| حسابات بكلمة مرور فارغة؟ | `/etc/shadow` | دخول مباشر بدون كلمة مرور |
| قواعد NOPASSWD في sudo؟ | `/etc/sudoers` + `/etc/sudoers.d/` | تصعيد صلاحيات بدون كلمة مرور |

> **ملاحظة مهمة:** إذا تعذّرت قراءة أي مصدر (تشغيل بدون `sudo` مثلاً)، فجميع فحوص المستخدمين الأربعة تُعيد `None` وتُعطي `WARN` لا `PASS` وهمي — تمييز كامل بين «لا مشكلة» و«تعذّر الفحص».

**الدوال الرئيسية:**

| الدالة | الوظيفة | تُعيد |
|--------|---------|-------|
| `audit_users()` | تُشغّل الفحوصات الأربعة | قائمة نتائج |
| `_root_locked()` | يتحقق من حالة root عبر `passwd -S` ثم `/etc/shadow` | `True/False/None` |
| `_uid0_non_root()` | يقرأ `/etc/passwd` ويبحث عن UID=0 | قائمة أسماء أو `None` إذا تعذّرت القراءة |
| `_empty_passwords()` | يقرأ `/etc/shadow` | قائمة أو `None` إذا محجوب |
| `_nopasswd_sudo()` | يقرأ كل ملفات sudoers | قائمة أو `None` إذا لا شيء مقروء |

## `scoring.py` — حساب الدرجات

**ماذا يفعل؟**

يأخذ نتائج جميع الفحوصات ويحسب درجة مئوية لكل نطاق، ثم يدمجها في درجة نهائية موزونة.

**الأوزان:**

```
الدرجة النهائية = (نسبة SSH × 40%) + (نسبة Firewall × 35%) + (نسبة Users × 25%)
```

**مثال:**

```
SSH:      60/65 = 92.3%  × 0.40 = 36.9
Firewall: 25/35 = 71.4%  × 0.35 = 25.0
Users:    25/25 = 100%   × 0.25 = 25.0
─────────────────────────────────────────
الدرجة النهائية = 86.9 / 100  →  Good
```

**ذكاء التطبيع:** إذا فحصت نطاقاً واحداً فقط (مثلاً `--audit ssh`)، تُطبَّع الأوزان تلقائياً حتى تبقى النتيجة من 100.

**الدوال:**

| الدالة | الوظيفة |
|--------|---------|
| `calculate_scores()` | يحسب الدرجات ويُعيد `(domain_scores, final_score)` |
| `score_label()` | يُحوّل رقم إلى نص (Good / Needs Improvement / Danger / Critical) |

## `exporter.py` — تصدير التقارير

**ماذا يفعل؟**

يُصدّر نتائج الفحص إلى ملفات خارجية للأرشفة أو المشاركة.

**صيغة JSON** — مناسبة للمعالجة البرمجية أو الدمج مع أنظمة SIEM:

```json
{
  "tool": "Siyaj — SecureConfig Auditor v1.0",
  "timestamp": "2026-05-17T14:30:22",
  "final_score": 86.9,
  "score_label": "Good",
  "domain_scores": { ... },
  "results": { ... }
}
```

**صيغة TXT** — مناسبة للقراءة البشرية والطباعة.

**الدوال:**

| الدالة | الوظيفة |
|--------|---------|
| `export_json()` | يُصدّر كل شيء بصيغة JSON منسّق |
| `export_txt()` | يُصدّر تقريراً نصياً منسقاً يدوياً |

كلاهما يلتقط `OSError` ويطبع رسالة خطأ بدل أن يُوقف البرنامج.

## `cis_rules.json` — قاعدة القواعد

**ماذا يفعل؟**

يخزّن كل قواعد CIS Benchmarks منفصلة عن الكود تماماً. هذا يعني:
- يمكن تعديل الأوزان أو إضافة قواعد جديدة **بدون لمس أي كود Python**
- يمكن مراجعة القواعد من شخص غير مبرمج

**هيكل كل قاعدة:**

```json
"PermitRootLogin": {
    "expected": "no",       ← القيمة الآمنة المطلوبة
    "weight": 15,           ← الوزن داخل النطاق
    "description": "...",   ← وصف الفحص
    "comparator": "lte"     ← اختياري: lte = أصغر أو يساوي
}
```

**توزيع الأوزان داخل كل نطاق:**

```
SSH (65 نقطة داخلياً):
  PermitRootLogin        15
  PasswordAuthentication 15
  Protocol               10
  X11Forwarding           5
  MaxAuthTries            5
  IgnoreRhosts            5
  HostbasedAuthentication 5
  PermitEmptyPasswords    5

Firewall (35 نقطة داخلياً):
  ufw_active             15
  default_incoming_deny  10
  dangerous_ports        10

Users (25 نقطة داخلياً):
  root_locked            10
  no_uid0_non_root        8
  no_empty_passwords      4
  no_nopasswd_sudo        3
```
